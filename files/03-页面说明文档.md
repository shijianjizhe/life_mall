# 人生模拟商城（Life Mall）—— 页面说明文档
> 版本：v2.0 · 共20个页面，含详细视觉风格规范

---

## 第一部分：整体设计系统（Design System）

在设计每个页面之前，先统一以下规范，保证20个页面风格一致。

### 1.1 品牌调性

扁平插画风 + 潮流梗文化 + 轻二次元质感，介于"小红书治愈系"和"抽象梗表情包"之间。**不走真实电商的严肃质感**，要让人一眼看出"这是个不正经但很好玩的东西"。

### 1.2 色彩系统

**主品牌色（用于Logo、主按钮、全局强调色）**：
- 主渐变：蜜桃粉 `#FF6B9D` → 亮紫 `#A855F7`（用于CTA按钮、Logo、加购物车按钮）
- 强调色（点缀/成功状态）：荧光青柠 `#B4FF39`
- 警示/清空类操作：珊瑚红 `#FF5A5F`

**基础背景与文字**：
- 主背景（浅色模式）：米白 `#FAF7F2`
- 主背景（深色模式，二期可选）：深灰蓝 `#16161D`
- 主文字：深灰紫 `#2D2D3A`
- 次要文字/说明文字：灰紫 `#8A8A9E`
- 分割线/卡片描边：浅灰 `#ECE8E1`

**四大分区专属主题色**（每个分类页面的背景色调、图标配色均使用对应主题色，形成强烈的"换了个世界"的视觉区分）：

| 分区 | 主题色 | 视觉感觉 |
|---|---|---|
| 😂 现实买不起 | 香槟金 `#D4AF37` + 深空黑 `#1A1A2E` | 奢华、精英感、略带反讽 |
| 🧸 小时候梦想 | 马卡龙青绿 `#4ECDC4` + 明黄 `#FFE66D` | 童趣、温暖、怀旧 |
| 🌕 不可能拥有 | 深邃紫蓝 `#2E1F5E` + 星光银 `#C0C0F0` | 梦幻、太空感、超现实 |
| ✨ 抽象商品 | 荧光绿 `#B4FF39` + 纯黑 `#0A0A0A` | 赛博、整活、梗文化感 |

### 1.3 字体系统

- 中文正文：思源黑体 / 阿里巴巴普惠体（清晰易读）
- 中文标题/口号型文案：可选用"站酷快乐体"或类似手写感梗字体，用于"订单成功""AI吐槽"这类需要情绪冲击力的场景
- 英文/数字：Poppins 或 Inter（圆润现代，价格数字使用 Bold/加粗大字号突出）
- 字号建议：H1 28-32px / H2 20-22px / 正文 14-16px / 辅助说明 12px

### 1.4 插画与图标风格

- 商品图：建议统一使用扁平插画风格生成（而非写实摄影图），弱化"真实购物"的严肃感，强化"玩具/梗"的娱乐感
- 图标：线性图标 + 偶尔点缀3D轻质感图标（如首页四个分区入口可用略带立体感的图标增加吸引力）
- 表情包元素：允许在文案中穿插 emoji（😂🎉🌕✨🐶）增强网络热梗氛围

### 1.5 动效原则

- 所有交互反馈使用"弹性缓动"（spring easing），避免生硬的线性动画
- 加入购物车：商品缩略图以抛物线轨迹"飞入"购物车图标，图标伴随轻微放大回弹
- 下单成功：全屏彩带/confetti动画，持续约1.5秒
- 按钮点击：轻微 scale(0.96) 反馈
- 页面切换：轻量的淡入+上滑（避免过度炫技影响性能）

### 1.6 文案语气基调

第一人称吐槽、网络热梗风格，大量使用"（）"内心OS式吐槽、emoji、"doge""笑不活了"等网络用语，但避免低俗和攻击性表达（详见PRD文档"文案与语气规范"章节）。

## 第二部分：核心购物流程页面

### 页面1：启动引导页 `/onboarding`

**目标**：首次打开时，用最快速度讲清楚"这是什么、怎么玩、数据存哪"，避免用户误解为真实电商。

**视觉风格**：
- 全屏渐变背景（主品牌色蜜桃粉→亮紫渐变），配合大幅可爱插画（如一个购物车漂浮在云朵/星星中）
- 采用"3屏轮播"卡片式引导，底部圆点指示器

**内容分屏**：
1. 第1屏：产品Logo + Slogan（"永远不会到货，但足够快乐"）+ 插画
2. 第2屏：玩法介绍（"逛 → 加购 → 下单 → 获得快乐"四个步骤图标+简短文字）
3. 第3屏：数据须知（"你的所有数据只保存在这台设备的浏览器里，记得常备份哦"）+ "开始逛"按钮

**交互**：
- 支持左右滑动切换/点击"下一步"
- 最后一屏点击"开始逛"→ 跳转首页，且本地记录"已完成引导"（下次不再展示，可在设置页重新查看）

### 页面2：首页 `/`

**目标**：营造"逛"的第一入口，快速吸引用户进入某个情绪分区。

**视觉风格**：
- 背景：米白色 `#FAF7F2`，顶部有一条渐变色装饰条
- 整体版式：类似"小红书首页 + 淘宝频道页"的混合感，卡片圆角较大（16px+），营造软萌感

**布局结构（从上到下）**：
1. **顶部导航栏**：左侧Logo文字"人生商城"，右侧头像入口 + 购物车图标（图标右上角红色数字角标，购物车有商品时角标带轻微呼吸动效）
2. **虚拟购物声明条**：浅黄色背景横幅，滚动展示"本站商品仅供娱乐，不会真实发货收款"文字，字号12px，不刺眼但清晰可见
3. **搜索入口**：一个圆角搜索框（占位文字："想买点让自己开心的东西？"），点击跳转搜索页
4. **四大分区入口卡片**：2x2 网格布局，每个卡片使用对应主题色背景 + 大emoji图标 + 分区名称，卡片有轻微阴影和悬浮态放大效果
5. **"今日推荐"横向滑动卡片流**：卡片包含商品图、名称、价格，顶部有一句AI/规则生成的引导语（如"今天适合买点让自己开心的东西"）
6. **专题活动入口横幅**："如果你有100万，你会怎么花？" 大banner，点击进入专题页
7. **底部固定导航栏**：首页 / 分类 / 购物车 / 我的（4个图标+文字，选中态使用主品牌色高亮）

**交互细节**：
- 点击分区卡片 → 转场动画跳转对应分类列表页
- 购物车图标角标随加购实时更新（弹跳动效，0.3秒）

### 页面3：分类列表页 `/category/:categoryCode`（4个分类复用此模板）

**目标**：让用户在特定情绪主题下持续"逛"，且每个分区要有强烈的差异化氛围感。

**视觉风格（以"现实买不起"为例，其余3个分类替换对应主题色和插画元素）**：
- 顶部大面积使用分区主题色作为背景（香槟金渐变到深空黑），有专属插画点缀（如金色皇冠、豪车剪影）
- 商品卡片区域切回米白底色，形成"顶部沉浸氛围 + 下方正常浏览"的层次感

**布局结构**：
1. **顶部氛围区**：分类大标题（如"😂 现实买不起"）+ 一句吐槽向副标题（如"看看就好，别当真"）+ 返回按钮
2. **筛选/排序条**：价格排序、"最想要排序"（Tab切换样式）
3. **商品网格列表**：2列卡片布局，每张卡片含：
   - 商品图（占卡片上方60%面积）
   - 商品名称（1行，超出省略）
   - 价格（加粗大字号，使用分区主题色）
   - 右下角"+"加入购物车圆形按钮（点击有飞入动效）
4. **上拉加载更多**（本地分页，每页加载12-16件商品）

**分区差异化要点**：
- 现实买不起：卡片描边细金线，字体略带"奢华感"
- 小时候梦想：卡片圆角更大更软，插画使用马卡龙色系
- 不可能拥有：背景加入星空/渐变紫色点缀，营造超现实感
- 抽象商品：卡片采用黑底荧光绿描边，赛博朋克风格字体点缀

**空/异常状态**：暂无该分类商品时展示"这个星球的商品还在路上"插画+文案

### 页面4：搜索页 `/search`

**目标**：快速定位商品，同时保留娱乐调性（不做成纯工具化搜索框）。

**视觉风格**：
- 顶部搜索输入框（自动聚焦），下方展示"猜你想搜"标签云（如"保时捷""乐高""好运"等热门关键词标签，圆角胶囊样式，随机配色取自四大分区主题色）

**布局结构**：
1. 顶部：搜索输入框 + 取消按钮
2. 搜索历史（本地存储，最多展示8条，支持清空）
3. 热门关键词标签云
4. 搜索结果：与分类列表页相同的商品网格样式

**交互**：输入时实时本地过滤商品名称（无需等待，纯前端瞬间响应，体现"丝滑"的体验优势）

---

### 页面5：商品详情页 `/product/:productId`

**目标**：制造"种草"沉浸体验，是激发"加购冲动"的关键页面。

**视觉风格**：
- 顶部大图轮播（占屏幕上方45%），背景使用该商品所属分类的主题色作为轮播底色
- 下方内容区回归米白底色

**布局结构**：
1. **顶图轮播区**：商品主图+多图，支持左右滑动，图片下方圆点指示器
2. **价格与名称区**：商品名称（20px加粗）+ 价格（28px超大加粗，使用主品牌色）+ 荒诞库存文案（如"库存：仅剩你的自尊心"，灰色小字，带点小揶揄）
3. **商品描述区**：预置/AI生成的幽默向文案，卡片式呈现，可折叠展开
4. **"买家秀"评论区**：3-5条虚构搞笑评论，采用聊天气泡/评论卡片样式，含虚构昵称+头像+"购买"时间戳
5. **底部固定操作栏**（吸底，白色背景+轻微阴影）：
   - 左侧：收藏心形图标（点击变红+缩放动效）
   - 中间：AI客服气泡入口（点击弹出/跳转AI聊天页）
   - 右侧两个主按钮："加入购物车"（描边按钮）+ "立即下单"（品牌渐变色实心按钮，视觉权重更高）

**交互**：
- 点击"加入购物车"→ 商品缩略图飞入购物车图标动效 + 轻微震动反馈（移动端）+ Toast提示"已加入购物车（这次是真的）"
- 点击AI客服气泡 → 底部抽屉弹起简易对话框，或直接跳转全屏AI聊天页（视内容长度决定，建议简短对话用抽屉、深度互动跳全屏页）

## 第三部分：情绪交互模块页面

### 页面6：AI客服全屏聊天页 `/ai-chat/:productId?`

**目标**：承载更深度的"情绪陪伴"互动，让用户感受到"有人在陪你逛"的拟人化体验。

**视觉风格**：
- 顶部仿聊天App导航栏："AI客服-小助手" + 一个卡通头像（圆形，风格统一为品牌吉祥物，可设计成一个戴着购物袋的可爱生物）
- 对话背景：浅米色，聊天气泡采用圆角矩形，AI气泡使用浅紫色背景，用户气泡使用品牌粉色背景

**布局结构**：
1. 顶部导航栏（头像+"在线"状态点缀，营造"真的有人"的错觉感，但整体仍需保持"这是AI"的透明度，不刻意误导）
2. 聊天记录区（气泡自下而上滚动）
3. 快捷回复标签（如"这个东西真的好看吗""帮我砍一刀（假的）""安慰我一下"），点击直接发送预置问题
4. 底部输入框 + 发送按钮

**交互**：
- 若关联了具体商品（从商品详情页跳入），AI首句自动围绕该商品展开对话（如"亲，这个真的很适合您，但是我们不会发货哦😊"）
- 默认模式下使用本地规则+预置文案库匹配关键词回复；若用户开启了"AI进阶模式"，则调用真实LLM API生成更灵活的回复

### 页面7：购物车页 `/cart`

**目标**：全产品情绪价值的核心承载页——"看到购物车爆炸的爽感"。

**视觉风格**：
- 背景保持米白基调，但购物车内商品数量较多时，顶部可触发"购物车已炸"的彩蛋横幅（荧光色背景+"你的购物车已突破天际🚀"文案），增强满足感反馈

**布局结构**：
1. **顶部标题栏**："购物车（共N件）"+ 一键清空入口（右上角小字，点击需二次确认）
2. **AI吐槽悬浮气泡**：页面顶部下方嵌入一条动态吐槽语，根据购物车商品分类关键词自动生成（如全是数码类 → "你这购物车，一看就是被工资偷走快乐的人"）
3. **商品列表**：每项卡片含缩略图、名称、单价、数量加减器（-/+按钮，中间数字）、删除按钮（左滑呼出，类似iOS列表交互）、勾选框
4. **底部结算栏**（吸底）：
   - 左侧：全选复选框 + "合计：¥XXX"（大号加粗品牌色数字）
   - 右侧："立即结算"按钮（品牌渐变色，若购物车为空则置灰不可点击）
5. **空购物车状态**：居中插画（一个空荡荡的购物车飘在云朵上）+ 文案"购物车空空如也，快去挑几个不用还房贷的东西吧" + "去逛逛"按钮

**交互**：
- 数量加减实时更新总价（无延迟，纯本地计算）
- 左滑商品卡片呼出删除按钮，滑动删除有淡出+高度塌陷动效

### 页面8：假结算/支付动画页 `/checkout`

**目标**：还原真实支付的仪式感，强化"下单"这个动作的重量感，即使全程是假的。

**视觉风格**：
- 背景采用轻微渐变（浅紫到浅粉），营造"庄重感与幽默感并存"的反差

**布局结构**：
1. **订单信息摘要卡片**：商品缩略图横向排列 + 数量 + 合计金额
2. **"收货地址"模块**：装饰性展示，可预置固定选项如"发往：我的心里""发往：内心深处的平行宇宙"，用户可选择或自定义输入（纯娱乐，不做实际用途）
3. **"支付方式"选择区**：搞笑虚拟选项列表（单选样式），例如：
   - 💰 花呗（假的）
   - 🪙 碎银子
   - 🌸 青春
   - 😅 下辈子还
4. **底部"确认支付"大按钮**（品牌渐变色，全宽）

**交互**：
- 点击"确认支付"→ 全屏Loading动画：一个旋转的卡通购物车图标 + 进度条 + 底部轮播搞笑文案（如"正在联系永远不会来的快递员……""正在假装扣款……"），持续约1.5-2.5秒
- 动画结束后自动跳转订单成功页（无需用户手动点击，保持"一气呵成"的爽感）

### 页面9：订单成功页 `/order/success/:orderId`

**目标**：产品的"梗点高潮"，制造会心一笑的记忆点，并引导分享/复访/备份数据。

**视觉风格**：
- 全屏彩带/confetti动画从顶部洒落，背景为品牌渐变色，营造"仪式感拉满"的反差萌

**布局结构**：
1. 大号动效图标（打勾/礼花）+ "订单成功！🎉"（超大加粗标题）
2. 核心梗文案区（从预置文案池随机展示，如"预计送达时间：永远不会（doge）"）
3. 订单摘要卡片（商品缩略图+总价+订单编号）
4. **首单专属提示卡片**（仅首次下单成功时出现）："第一单达成！建议现在就备份一下你的购物车小宇宙~" + "去备份"按钮
5. 操作按钮组（横向三个次级按钮）："生成分享海报" / "继续逛" / "查看历史订单"

## 第四部分：沉淀与个人内容页面

### 页面10：历史订单列表页 `/orders`

**视觉风格**：简洁列表风格，与购物车页保持一致的卡片样式，但整体更"档案化"（背景纯白，减少花哨装饰，方便快速浏览历史）

**布局结构**：
1. 顶部标题"我的订单"+ 订单总数统计
2. 订单卡片列表（每项含：下单时间、商品缩略图集合、总价、"预计送达"梗文案），点击进入订单详情（复用订单成功页样式，去掉彩带动效）
3. 空状态："你还没有下过单，快去剁手（虚拟）吧"

**布局结构**：

4. 空状态插画+文案："还没有心动的商品，去逛逛看有没有一见钟情的"

### 页面12：梦想清单页 `/wishlist`

**目标**：区别于购物车的"即时冲动"，承载"长期心愿"，情感浓度更高。

**视觉风格**：
- 背景采用深邃紫蓝渐变（呼应"不可能拥有"分区的梦幻感），星光点缀装饰，营造"许愿"的仪式氛围

**布局结构**：
1. 顶部标题"梦想清单 ✨"+ 副标题"写下你想拥有的一切，不管多离谱"
2. 清单条目列表：每项为一张卡片，可关联已有商品（展示商品图）或纯文字自定义愿望（如"环游世界""买套海景房"），左侧有一个星星/爱心图标
3. 底部"+ 添加新愿望"按钮 → 弹出输入框（文字输入 或 从商品库选择关联）
4. 每个条目支持"划掉/标记已实现"（虚拟意义上的自我鼓励机制），划线动效+彩纸屑小庆祝

### 页面13：虚拟房间布置页 `/room`

**目标**：将"已下单"的商品进行可视化摆放，制造"我拥有了"的具象化满足感，同时具备较强的把玩性和截图分享属性。

**视觉风格**：
- 房间背景采用暖色调2D等距（isometric）或平面卡通插画风格房间场景，用户可选择不同房间背景主题（如"简约风""太空舱""复古客厅"，二期可扩展更多主题解锁玩法）

**布局结构**：
1. 顶部：房间主题切换入口 + "保存/分享"按钮
2. 中间：房间画布区域（占屏幕主要面积），用户已"购买"的商品以缩略图形式可拖拽摆放在场景中
3. 底部：横向滑动的"已购商品素材栏"（拖拽到画布使用），未使用的商品灰度显示，已摆放的商品高亮
4. 画布内每个已摆放物品支持：拖动位置、双指缩放（移动端）/滚轮缩放（桌面端）、点击选中后出现删除/调整层级的小工具条

**交互**：拖拽有轻微磁吸对齐辅助线，摆放成功有轻微震动反馈

---

### 页面14：海报生成/分享页 `/share/poster`

**目标**：一键生成适合发朋友圈/小红书的"战绩截图"，是产品最大的社交传播入口。

**视觉风格**：
- 提供2-3套海报模板供选择（如"购物车画像风""年度账单风""梦想清单风"），每套模板配色对应不同分区主题色系

**布局结构**：
1. 顶部：模板切换Tab（横向滑动选择）
2. 中间：海报实时预览区（基于用户购物车/收藏/订单数据自动填充拼贴图+标题文案，如"我的年度购物车画像：数码荷尔蒙过盛型"）
3. 底部操作栏："保存图片到本地"按钮（主按钮）+ "分享"按钮（调用系统分享API，若浏览器不支持则降级为复制图片/长按保存提示）

### 页面15：AI人格报告详情页 `/ai-report`

**目标**：基于用户历史购物行为生成一份幽默人格画像，制造"被看穿"的惊喜感和分享欲。

**视觉风格**：
- 整体设计参考"网易云年度报告""MBTI测试结果页"的仪式感排版，纵向卡片式滚动，每滑一屏揭晓一个维度的"人格解读"

**布局结构**（分屏滚动展示）：
1. 第1屏：封面——"你的购物人格报告"+ 加载动效（"正在分析你的购物车灵魂……"）
2. 第2屏：人格标题揭晓（大号标题，如"数码荷尔蒙过盛型购物人格"）+ 配图（拟人化插画角色）
3. 第3屏：数据支撑（"你加购最多的分类是：XX""你最疯狂的一次加购了XX件商品"）
4. 第4屏：一段AI/预置模板生成的幽默点评文字
5. 底部："生成分享图" + "重新测一次（基于最新数据）"按钮

## 第五部分：账户、数据与设置页面

### 页面16：个人中心页 `/profile`

**目标**：沉淀用户身份感和长期数据总览，是通往各类二级功能的枢纽。

**视觉风格**：与首页保持同源的品牌渐变风格，但信息密度更高、更"仪表盘化"

**布局结构**：
1. **用户信息卡**：头像（可本地上传或选择预置卡通头像）+ 昵称（本地可编辑）+ "加入第N天"标签
2. **数据总览区**（3-4个数据卡片横向排列）：累计"下单"次数、累计"消费"金额（虚拟）、收藏数、连续签到天数
3. **功能入口列表**（图标+文字的列表项）：
   - AI购物人格报告
   - 历史订单
   - 梦想清单
   - 虚拟房间
   - 数据备份与恢复
   - 设置
4. 每个入口项右侧带小箭头，点击跳转对应页面

---

### 页面17：数据备份与恢复页 `/backup`

**目标**：本产品最重要的"保险丝"页面，必须让用户清晰理解且愿意主动使用。

**视觉风格**：区别于其他页面的活泼感，本页采用更沉稳的配色（浅蓝灰背景），传递"这是个正经功能"的信号，降低用户的操作顾虑

**布局结构**：
1. 顶部说明卡片："你的所有数据（购物车、订单、收藏等）都只保存在这台设备的浏览器里，清除浏览器数据会导致丢失。建议定期备份！"
2. **导出数据**模块："导出为文件"大按钮，点击后触发浏览器下载一个 `.json` 备份文件，成功后有Toast提示+图标动效
3. **导入/恢复数据**模块："选择备份文件"按钮，选择文件后弹出二次确认弹窗（"导入将覆盖当前所有本地数据，确认继续吗？"），确认后展示恢复进度和结果提示
4. 底部小字：显示"上次备份时间"（若有记录）

---

### 页面18：设置/关于/免责声明页 `/settings`

**目标**：承载账户级设置、AI进阶模式开关、关于信息与免责声明。

**视觉风格**：标准列表式设置页，简洁克制，减少视觉干扰，突出可读性

**布局结构（分组列表）**：
1. **通用设置组**：昵称修改、头像修改、深色模式开关（若支持）、重新查看启动引导
2. **AI设置组**：
   - "AI进阶模式"开关（默认关闭）
   - 开启后展示 API Key 输入框 + 明显的风险提示文案（"密钥仅保存在本设备浏览器，请勿在公共/共享设备开启此功能"）
3. **数据管理组**：跳转"数据备份与恢复"入口、"清空所有本地数据"危险操作（珊瑚红文字，需二次确认+输入确认文字才可执行）
4. **关于我们组**：
   - 产品介绍
   - **虚拟购物声明**（完整版法律/免责声明文字，需明确注明"本产品所有交易均为虚拟娱乐，不涉及真实支付、发货及售后服务"）
   - 版本号

## 第六部分：活跃与专题页面

### 页面19：每日签到页 `/checkin`

**目标**：建立日常打开习惯，提供轻量正反馈。

**视觉风格**：明亮活泼，采用日历卡片式布局，签到成功当日的格子会有"贴纸"式的可爱图标点亮效果

**布局结构**：
1. 顶部："连续签到 N 天" 大号数字展示 + 快乐币余额图标
2. 本月签到日历网格（7列，已签到日期高亮显示贴纸图标，未签到为灰色空格，今天有呼吸光效提示可签到）
3. 中间："签到领快乐币"大按钮（未签到时为品牌渐变色可点击态，已签到后变灰并显示"明天再来"）
4. 底部：连续签到奖励说明（如"连续7天额外奖励""连续30天解锁隐藏房间主题"）

**交互**：点击签到按钮 → 按钮弹跳动效 + 快乐币数字滚动增加动画 + 小型confetti反馈

---

### 页面20："如果你有100万"专题活动页 `/event/million`

**目标**：产品最强的传播型专题内容，制造话题讨论度和分享欲望。

**视觉风格**：
- 采用金色奢华质感背景（呼应"现实买不起"分区但更强化仪式感），顶部大标题采用烫金字体效果

**布局结构**：
1. **顶部预算展示区**：巨大数字"¥1,000,000.00" + 副标题"如果给你100万，你会怎么花？"
2. **实时预算进度条**：随用户选购商品扣减，进度条颜色随比例变化（充裕时为绿色，接近超支时变橙红色）
3. **商品选购区**：复用商品网格卡片样式，但操作变为"加入清单"而非"加入购物车"（数据独立，不与常规购物车混淆）
4. **底部结算区**：
   - 剩余预算数字实时展示
   - 若超支：弹出幽默提示"就知道你managing money能力不行😂，但没关系反正是假钱"
   - 若花光预算（0-1000元误差内视为"完美花光"）：触发彩蛋动画+"财富自由挑战达成！"称号
5. **生成专题海报**按钮：将本次"100万清单"生成专属分享海报，海报模板与常规海报生成页视觉区分（金色主题）
